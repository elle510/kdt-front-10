CSS 우선순위 3가지 원칙

1. 후자 우선의 원칙 (Cascade)

같은 우선순위라면 나중에 작성된 스타일이 적용됩니다.

.cascade-test { color: red; /* 먼저 작성 */ }
.cascade-test { color: blue; /* 나중에 작성 - 이것이 적용됨 */ }
이 텍스트는 파란색입니다 (나중에 작성된 blue가 적용됨)
이 텍스트는 보라색이고 밑줄이 있습니다

2. 구체성의 원칙 (Specificity)

더 구체적인 선택자가 우선 적용됩니다.

/* 구체성 점수 계산 */
p { } /* 점수: 1 (타입) */
.class { } /* 점수: 10 (클래스) */
#id { } /* 점수: 100 (ID) */
#id .class p { } /* 점수: 111 (ID + 클래스 + 타입) */

이 텍스트는 ID 선택자(파란색)가 적용됩니다

이 텍스트는 가장 구체적인 선택자(초록색, 굵게)가 적용됩니다

이 박스는 #content .box 선택자가 적용됩니다 (분홍 배경, 빨간 테두리)
선택자 구체성 점수 적용 여부
.box 10 ❌
div.box 11 ❌
.container .box 20 ❌
#content .box 110 ✅ 적용됨

3. 중요성의 원칙 (!important)

!important가 모든 우선순위를 무시하고 최우선 적용됩니다.

.test { color: red; } #id { color: blue; /* 구체성이 더 높음 */ }
.override { color: white !important; /* 모든 것을 무시하고 적용됨 */ }
이 텍스트는 !important 때문에 흰색 배경에 보라색입니다
!important끼리는 후자 우선 원칙이 적용됩니다 (진한 초록색)
⚠️ !important 사용 주의사항:
  • 디버깅을 어렵게 만듭니다
  • CSS의 자연스러운 흐름을 깨뜨립니다
  • 정말 필요한 경우에만 사용하세요

CSS 우선순위 전체 순서 (높은 순서대로)

  1. 인라인 스타일 + !important
  2. ID 선택자 + !important
  3. 클래스 선택자 + !important
  4. 타입 선택자 + !important
  5. 인라인 스타일 (style="...")
  6. ID 선택자 (#id)
  7. 클래스 선택자 (.class), 속성 선택자 ([attr])
  8. 타입 선택자 (element)
  9. 전체 선택자 (*)

기억하세요: 같은 우선순위라면 나중에 작성된 것이 적용됩니다!
