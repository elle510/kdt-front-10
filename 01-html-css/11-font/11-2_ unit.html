CSS 단위 완전 가이드

브라우저 창 크기를 조절하면서 각 단위의 차이점을 확인해보세요!

px (픽셀) - 절대 단위

화면 크기에 상관없이 항상 같은 크기를 유지합니다.

100px 너비, 16px 글씨
200px 너비, 20px 글씨
300px 너비, 24px 글씨

특징: 정확한 크기 제어, 반응형에는 부적합

% (퍼센트) - 부모 요소 기준

부모 요소 크기의 백분율로 계산됩니다.

50% 너비, 50% 높이
75% 너비, 25% 높이
100% 너비, 10% 높이

특징: 부모 크기에 따라 변함, 레이아웃에 유용

vw (Viewport Width) - 뷰포트 너비 기준

브라우저 창 너비의 1% = 1vw

25vw (화면 너비의 25%)
50vw (화면 너비의 50%)
100vw (화면 너비의 100%)

특징: 화면 크기에 따라 변함, 반응형 디자인에 유용

vh (Viewport Height) - 뷰포트 높이 기준

브라우저 창 높이의 1% = 1vh

20vh
30vh

특징: 화면 높이에 따라 변함, 풀스크린 요소에 유용

vmin / vmax - 뷰포트 최소/최대값 기준

vmin: vw와 vh 중 작은 값 | vmax: vw와 vh 중 큰 값

20vmin (정사각형)
15vmax

특징: 화면 비율에 관계없이 일정한 비율 유지

em - 부모 요소 폰트 크기 기준

부모 요소의 font-size를 1em으로 계산 (현재 부모: 20px)

1em = 20px (폰트, 너비, 높이 모두)
1.5em = 30px
2em = 40px

특징: 중첩될 때마다 계산이 복잡해짐, 컴포넌트 내부 스케일링에 유용

rem - 루트 요소(html) 폰트 크기 기준

html 요소의 font-size를 1rem으로 계산 (현재: 16px)

1rem = 16px
1.5rem = 24px
2rem = 32px

특징: 일관된 계산, 전역 스케일링에 유용

단위별 사용 권장사항

px (픽셀)

  • 테두리 두께
  • 작은 간격
  • 정확한 크기가 필요한 경우

% (퍼센트)

  • 너비 설정
  • 레이아웃 구성
  • 부모 기준 크기 조절

vw/vh

  • 풀스크린 요소
  • 반응형 타이포그래피
  • 뷰포트 기준 레이아웃

rem

  • 폰트 크기
  • 일관된 간격
  • 전역 스케일링
