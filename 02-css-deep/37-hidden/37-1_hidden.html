CSS로 콘텐츠 숨기는 5가지 방법

1. display: none vs visibility: hidden

가장 기본적인 숨김 방법입니다.

이 요소 다음에 display: none이 적용된 요소가 있지만, 공간을 차지하지 않아 보이지 않습니다.
이 텍스트는 보이지 않습니다.
이 요소 앞에는 visibility: hidden이 적용된 요소가 있어, 보이지 않지만 자신의 공간은 그대로 차지하고 있습니다.
이 텍스트는 보이지 않습니다.
앞 요소의 공간 때문에 이 요소는 아래로 밀려났습니다. 
.hide-display { display: none; }
.hide-visibility { visibility: hidden; }
  • display: none: 요소를 렌더링 트리에서 완전히 제거합니다. 공간을 차지하지 않으며, 스크린 리더도 읽지 않습니다.
  • visibility: hidden: 시각적으로만 숨깁니다. 요소의 공간은 그대로 유지되며, 스크린 리더도 대부분 읽지 않습니다.

2. width: 0; height: 0;

너비와 높이를 0으로 만들어 시각적으로 보이지 않게 합니다. overflow: hidden;을 함께 사용해야 합니다.

이 요소 다음에 크기가 0인 요소가 있습니다.
이 텍스트는 보이지 않습니다.
앞 요소가 공간을 차지하지 않아, 이 요소는 바로 위에 붙어 있습니다. 
.hide-dimensions {
  width: 0; height: 0; overflow: hidden;
}
  • 공간을 차지하지 않으며, 스크린 리더가 읽지 않는 경우가 많습니다. display: none과 유사한 효과를 냅니다.

3. text-indent: -9999px;

주로 배경 이미지로 대체할 텍스트를 숨기기 위해 사용되던 오래된 기법입니다.

이 텍스트는 화면 밖으로 들여쓰기 되어 보이지 않습니다. 
.hide-text-indent {
  text-indent: -9999px;
  overflow: hidden;
  white-space: nowrap;
}
  • 요소의 공간은 그대로 차지합니다.
  • 접근성 측면에서 문제가 있을 수 있고, 최신 CSS 기법이 아니므로 사용을 권장하지 않습니다.

4. 화면 밖으로 보내기

시각적으로는 완전히 숨겨지지만, 스크린 리더는 정상적으로 읽을 수 있어 접근성에 유리한 방법입니다. "Skip to main content" 링크 등에 널리 사용됩니다.

이 텍스트는 화면에 보이지 않지만, 스크린 리더는 읽을 수 있습니다.

위 요소는 시각적으로 보이지 않지만, 페이지 소스나 개발자 도구에는 존재하며 스크린 리더 사용자에게 정보를 제공합니다. 

.sr-only { /* sr-only: screen-reader-only */
  position: absolute;
  left: -9999px;
  top: auto;
  width: 1px;
  height: 1px;
  overflow: hidden;
}
  • 용도: 스크린 리더 사용자에게는 정보를 제공해야 하지만, 디자인상 화면에는 보이지 않아야 하는 텍스트를 숨길 때 사용합니다.

5. CSS 클립 (⭐ 접근성을 위한 추천 방법)

시각적으로는 완전히 숨겨지지만, 스크린 리더는 정상적으로 읽을 수 있어 접근성에 가장 유리한 방법입니다. "Skip to main content" 링크 등에 널리 사용됩니다.

이 텍스트는 화면에 보이지 않지만, 스크린 리더는 읽을 수 있습니다.

위 요소는 시각적으로 보이지 않지만, 페이지 소스나 개발자 도구에는 존재하며 스크린 리더 사용자에게 정보를 제공합니다. 

.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  margin: -1px;
  padding: 0;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  clip-path: inset(50%);
  border: 0;
}
  • 용도: 스크린 리더 사용자에게는 정보를 제공해야 하지만, 디자인상 화면에는 보이지 않아야 하는 텍스트를 숨길 때 사용합니다.
