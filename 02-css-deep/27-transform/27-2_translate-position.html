transform: translate() vs. position

1. 시각적 배치 비교

두 방법 모두 요소를 원래 위치에서 시각적으로 이동시키지만, 원래 차지하던 공간은 그대로 유지합니다. 따라서 뒤따르는 sibling 요소의 위치는 영향을 받지 않습니다.

Position (relative)
Moved
Sibling
Transform (translate)
Moved
Sibling

2. 성능 및 렌더링 차이 (가장 중요한 차이점)

position의 top, left 값을 변경하면 브라우저가 Layout(Reflow)을 다시 계산해야 할 수 있어 성능에 부담을 줍니다. 반면, transform은 Composite 단계에서만 처리되어 GPU 가속을 받기 때문에 훨씬 부드러운 애니메이션을 제공합니다.

Position 애니메이션 (성능이 상대적으로 떨어짐)
Transform 애니메이션 (성능이 더 좋고 부드러움)

결론: 언제 무엇을 쓸까?

  • position: 페이지의 초기 **정적 레이아웃**을 구성할 때 사용합니다.
  • transform: 요소의 위치를 **동적으로 변경**하거나 **애니메이션**을 적용할 때 사용합니다. 성능상 이점이 명확합니다.
